Workflow Philosophy

"It's all about hands"

Problem Prediction

The real challenge in webdesign is not solving a problem but finding a strategy for an issue that uses don’t know they have yet. It stands for predicting users’ needs through Data and observation.

Experience Enhancement

Behind every smart solution there’s research and analysis : in order to improve User Experience, Webdesigners and developers have to build a solid website structure but also look for details that can make the difference.

Strategy Sketch

Now, after research and Data collection, we have all the tools to find a smart solution for users. It consists on designing a web product that could be represent what average user wants and expects.

Developed Derivation

The development of a website, derivated from its design, is the last but not the final phase of this process.  Yes, it’s about coding.  But bringing a project to life means to let it live as long as possible through tests, updates, reviews and so on.

Projects

Fanhub

#Team Project
#Javascript
#Node.js

A Fan Films’ streaming Website created during an intentive week at HETIC (Web School). I coded the Front-end features in Javascript and the upload film section in Node.JS.

P.P.

Even though users easily find fan-made films on the net, there’s no existing plateform which regroup all fan films.

E.E.

Categories, tags, a searchbar and a hamburger menu are available in order to let the user easily find a fan film.

S.S.

Fanhub has been conceived to be a one-page website where users access to film through an overlay which contains all film details and the video player.

D.D.

The website is dynamic thanks to Javascript data requests that make fanhub modular and optimised., such as template.

Jelly Dodge

#Team Project
#Javascript

Brick game created during a sleepless night at HETIC. We were a team composed by three designers and two JS developers. The other developer and I coded the game in Javascript using Oriented Object Programmation.

P.P.

Javascript-coded games are known to be not very optimized despite web games are widespread. This is why the real challenge was to create a bug-free funny game.

E.E.

User plays through keyboard using typical gameplay keys such as arrows and no mouse is needed so that game can be played on laptops too (as trackpads might not be so practical).

S.S.

In order to make the game more challenging, fishes-falling’s speed increases after a certain amount of time and so on.

D.D.

Jelly dodge is a Javascript game coded through Canvas API which allows web graphics rendering. Oriented Object Programmation has been used to improve performances.

Redesign Nespresso

#Solo Project
#UX/UI

UX Design project that consists on improving User Experience of Nespresso Website.  Wireframe was created on Sketch and protoyped on InVision.

P.P.

Current Nespresso Website is not well structured and seems quite messy, which makes user experience confusing. This may discourage user to buy Nespresso products online.

E.E.

Before accessing Nespresso website, a test is proposed in order to find the most suitable product for user.  The main purpose is making user experience more personal and interactive.

S.S.

The best strategy was giving a lighter design to Nespresso and its customers, using colors that remind coffee shades.  Moreover, many call-to-actions allow the user to easily find and buy a product.

D.D.

All user interactions are prototyped on InVision. As every single possible choice has its own Wireframe, User Experience has been schematized through a flowchart.
